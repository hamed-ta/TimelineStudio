# Startup Workflow

Use this workflow when the user says `/startup` or asks to resume work.

1. Read `AGENTS.md`.
2. Read `docs/handoff.md`.
3. Read `docs/plan.md`.
4. Read `docs/product.md`.
5. Skim relevant ADRs in `docs/adr/`.
6. Check repository status with `git status --short`.
7. Review recent commits with `git log --oneline -5`.
8. Summarize:
   - current goal
   - latest known state
   - open issues
   - last verification
   - next safe step

Do not edit files during startup unless the user asks.
