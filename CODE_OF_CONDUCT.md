# Code of Conduct

## Our Pledge

We want Timeline Studio to be a practical, respectful open source project. Contributors and maintainers are expected to keep discussions focused on the work, treat people with respect, and make the project approachable for users with different backgrounds and experience levels.

## Expected Behavior

- Use welcoming and inclusive language.
- Be respectful of differing viewpoints and constraints.
- Give and receive technical feedback constructively.
- Focus criticism on code, design, behavior, documentation, and process.
- Accept maintainer decisions when a discussion has reached a clear conclusion.

## Unacceptable Behavior

- Harassment, insults, threats, or discriminatory language.
- Sexualized language or imagery.
- Personal attacks or sustained disruption of project discussions.
- Publishing private information without explicit permission.
- Any conduct that would reasonably make the project unsafe or hostile for other participants.

## Scope

This code of conduct applies in project spaces, including issues, pull requests, discussions, release comments, and community channels connected to Timeline Studio. It also applies when someone represents the project in public.

## Reporting

If you experience or witness unacceptable behavior, report it to the project maintainers. If no private contact path is available, open a short public issue asking for a private maintainer contact and avoid including sensitive personal details.

Maintainers should review reports fairly, protect privacy where possible, and take action proportional to the situation.

## Enforcement

Maintainers may remove comments, close discussions, reject contributions, temporarily restrict participation, or permanently ban participants when behavior violates this code of conduct.

## Attribution

This policy is inspired by common open source community standards, including the Contributor Covenant, but is written specifically for Timeline Studio.
